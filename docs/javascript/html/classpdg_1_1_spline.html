Detailed Description

Curved path defined by control points, supporting multiple spline types including cubic Bezier curves.

Splines are mathematical curves defined by control points. PDG supports several spline types:

  • spline_Hermite - Hermite spline using position and tangent vector parameters
  • spline_Cardinal - Cardinal (Catmull-Rom) spline that passes through all control points
  • spline_UniformB - Uniform B-spline for smooth curves
  • spline_CubicBezier - Cubic Bezier curves with control handles
  • spline_TCB - Tension-Continuity-Bias spline for animation curves
  • spline_NaturalCubic - Natural cubic spline with continuous second derivatives

Hermite Spline Parameters

When using spline_Hermite, the four parameters to addSegment() are:

  • p1 - Start position (Point)
  • p2 - End position (Point)
  • p3 - Tangent vector at start (Point used as a direction vector, not a position)
  • p4 - Tangent vector at end (Point used as a direction vector, not a position)
Note
The tangent parameters (p3, p4) are direction vectors, not positions. They control the direction and speed of the curve as it leaves the start point and approaches the end point.
Hermite Example:
var spline = new pdg.Spline(pdg.spline_Hermite);
spline.addSegment(
new pdg.Point(10, 10), // p1: start position
new pdg.Point(100, 50), // p2: end position
new pdg.Point(30, 0), // p3: tangent vector (direction/speed leaving start)
new pdg.Point(30, 0) // p4: tangent vector (direction/speed approaching end)
);

◆ addPoint()

addPoint ( Point point)

adds a control point to the spline

This low-level method adds a single control point to the spline. The meaning and usage of control points depends on the spline type:

Hermite Splines (spline_Hermite):
Points are added in groups of 4: position1, position2, tangent_vector1, tangent_vector2. For Hermite splines, tangent points (3rd and 4th in each group) should be direction vectors, not positions. Use addSegment() instead for easier Hermite spline creation.
Cubic Bezier Splines (spline_CubicBezier):
Points are added in groups of 4: start, control1, control2, end. Additional segments share the endpoint of the previous segment.
Cardinal/Catmull-Rom Splines (spline_Cardinal):
Each point is a position that the curve passes through. The curve is calculated using a sliding window of 4 consecutive points.
Uniform B-Splines (spline_UniformB):
Each point is a control point. The curve approximates the shape defined by these points but doesn't necessarily pass through them.
Note
For most use cases, addSegment() provides a more intuitive interface, especially for Hermite and Bezier splines. Use addPoint() when you need fine-grained control over individual points or when building multi-segment Cardinal or B-splines.
Parameters
pointthe control point to add (interpretation depends on spline type)

addSegment ( Point p1,
Point p2,
Point p3,
Point p4 )

adds a spline segment with four control points

The meaning of the four points depends on the spline type:

Hermite Splines (spline_Hermite):
  • p1 - Start position (Point)
  • p2 - End position (Point)
  • p3 - Tangent vector at start (NOT a position, but a direction vector)
  • p4 - Tangent vector at end (NOT a position, but a direction vector)
Warning
For Hermite splines, p3 and p4 are tangent vectors (directions), not positions. They control the direction and speed of the curve at the start and end points. To create identical curves that are offset in position, use the same tangent vectors with different start/end positions.
Cubic Bezier Splines (spline_CubicBezier):
  • p1 - Start position
  • p2 - First control point (handle)
  • p3 - Second control point (handle)
  • p4 - End position
Cardinal/Catmull-Rom Splines (spline_Cardinal):
  • p1, p2, p3, p4 - Four consecutive control points
  • The curve passes through p2 and p3, using p1 and p4 to calculate tangents
Other Spline Types:
The four points serve as control points according to each spline type's mathematical definition.
Example - Hermite with tangent vectors:
var spline = new pdg.Spline(pdg.spline_Hermite);
spline.addSegment(
new pdg.Point(0, 0), // p1: start position
new pdg.Point(100, 50), // p2: end position
new pdg.Point(30, 20), // p3: tangent vector (go right 30, up 20)
new pdg.Point(30, -10) // p4: tangent vector (arrive going right 30, down 10)
);
Example - Cubic Bezier:
var bezier = new pdg.Spline(pdg.spline_CubicBezier);
bezier.addSegment(
new pdg.Point(0, 0), // start point
new pdg.Point(25, 75), // first control handle
new pdg.Point(75, 25), // second control handle
new pdg.Point(100, 100) // end point
);
