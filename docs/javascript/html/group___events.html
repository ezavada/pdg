◆ pdg::IEventHandler

class pdg::IEventHandler

Interface for any class which handles events.

Specific Subclasses should be written to implement particular handlers. Typically you would call an onXXX() method of the EventEmitter object that is generating the event you want, and it will take care of creating the event handler for you, and adding a convenient cancel() method to the handler. For example:

var mySpriteCollisionHandler = mySprite.onCollideSprite(function(event) {
// do something about the collision
});
...
mySpriteCollisionHandler.cancel(); // stop handling collision events with this handler

This cancel() method will only work for a single emitter. So if you may want to create a handler that is used with multiple emitters, you should create it once and then explicitly add it to and remove it from each emitter. Here's how:

var mySpriteHandler = new pdg.IEventHandler(function(event) {
console.log("in my event handler with "+event);
});
mySprite.addHandler(mySpriteHandler, pdg.eventType_SpriteCollide);
...
mySprite.removeHandler(mySpriteHandler, pdg.eventType_SpriteCollide);

◆ pdg::KeyEvent

struct pdg::KeyEvent

a key down or up event

Generated by the EventManager when the user pushes down on a key (pdg.eventType_KeyDown) or releases a key (pdg.eventType_KeyUp). Use this for arcade style key handling.

For keyboard data entry, use pdg.eventType_KeyPress. 

{ 
    emitter: {},       // the emitter that generated this event
    eventType: 4,      // the event type (eventType_KeyUp or eventType_KeyDown)
    keyCode: 34        // the raw key code from the OS
}
Platform-Specific Key Codes

The keyCode field contains platform-specific raw hardware keycodes from the operating system:

  • Mac OS X: Hardware scan codes from NSEvent (e.g., 56 for left shift)
  • Windows: Virtual key codes (VK_* constants, e.g., 0xA0 for left shift)
  • Linux/GLFW: GLFW scan codes
Warning
If you need to check for specific keys in a cross-platform way, use pdg.eventType_KeyPress events instead, which provide unicode character codes. However, modifier keys (shift, control, alt, meta) do not generate KeyPress events, so PDG provides platform-specific constants (see Key Code Constants) that allow you to write cross-platform code for detecting modifier key presses:
  • pdg.keyCode_LeftShift, pdg.keyCode_RightShift, pdg.keyCode_Shift
  • pdg.keyCode_LeftControl, pdg.keyCode_RightControl, pdg.keyCode_Control
  • pdg.keyCode_LeftAlt, pdg.keyCode_RightAlt, pdg.keyCode_Alt
  • pdg.keyCode_LeftMeta, pdg.keyCode_RightMeta, pdg.keyCode_Meta

◆ pdg::MouseEvent

struct pdg::MouseEvent

the user did something with the mouse

Generated by the EventManager whenever the user moves the mouse (pdg.eventType_MouseMove) or clicks a mouse button (pdg.eventType_MouseDown and pdg.eventType_MouseUp).

The EventManager also tracks the state of the mouse buttons, so you can retrieve that at any time with pdg.EventManager.isButtonDown(). The GraphicsManager tracks the position of the mouse, and that can be retrieved at any time with pdg.GraphicsManager.getMouse(). 

{ 
    emitter: {},            // the emitter that generated this event
    eventType: 7,           // the event type (eventType_MouseUp|Down|Move)
    shift: false,           // true if the shift key is held down
    ctrl: false,            // true if the control key is held down
    alt: false,             // true if the alt (or option) key is held down
    meta: false,            // true if the meta (windows or command) key is held down
    mousePos: {
        x: 456,                         // the x coordinate of the mouse when the event occurred
        y: 201              // the y coordinate of the mouse when the event occurred
    },
    leftButton: false,      // true if the left mouse button was down
    rightButton: false,     // true if the right mouse button was down
    buttonNumber: 0,        // button number (0=left, 1=right, 2=middle) or finger ID for touch
    lastClickPos: { 
        x: 456,                         // the x coordinate of the last mouseDown event
        y: 201              // the y coordinate of the last mouseDown event
    },
    lastClickElapsed: 201   // the number of milliseconds since the last mouseDown
}

Location and time since the last mouseDown event are provided to make it easier to detect double-clicks, dragging, and gestures.

Note
on a touch device mouse moved events are only generated when the user is touching the device, and each finger touching is treated like a separate mouse button, with the button number based on the order the fingers touched the device.
A SpriteLayer has the ability to automatically handle these mouse events and generate pdg.eventType_SpriteTouch for any Sprite that was clicked on, or pass these on as mouse events if no Sprite was hit.

◆ pdg::SpriteAnimateEvent

struct pdg::SpriteAnimateEvent

a Sprite did some animation (Optional)

A Sprite will generate one of these events when it completes an animation including fades (opacity animations), and moving on/offscreen or outside of the layer.

For frame based animations – that is, calls to pdg.Sprite.startFrameAnimation() – if it reaches the last frame and is not set to loop, action will be pdg.action_AnimationEnd. For looping animations pdg.action_AnimationLoop will be received each time the animation completes and starts over with the first frame.

When a Spriter animation blend – started by pdg.Sprite.blendToAnimation() – completes, action will be pdg.action_AnimationBlendComplete

For Spriter triggers embedded in a Spriter animation, action will be pdg.action_SpriterTrigger

For fades, one of three action types are possible: pdg.action_FadeComplete for calls to pdg.Sprite.fadeTo(), pdg.action_FadeInComplete for calls to pdg.Sprite.fadeIn(), and pdg.action_FadeOutComplete for calls to pdg.Sprite.fadeOut().

When a Sprite has setWantsOffscreenEvents(true), pdg.action_Offscreen and pdg.action_Onscreen events will be generated for that sprite whenever it enters or departs the visible area of the port the layer is being rendered into.

When a Sprite has setWantsCollideWallEvents(true), pdg.action_ExitLayer events will be generated for that sprite whenever it moves completely outside the boundaries of the layer. (It will also get a SpriteCollideEvent when it hits the boundary). 

{
    emitter: {},       // the emitter that generated this event
    eventType: 17,     // the event type (eventType_SpriteAnimate)
    action: 0,         // what happened (action_AnimationEnd/Loop or action_Fade/In/OutComplete)
    actingSprite: {},  // the Sprite that was animating
    inLayer: {}        // the SpriteLayer that contains the Sprite
}
Note
At this time other kinds of animations such as calls to pdg::Sprite.startAnimation() do not generate any events.

◆ pdg::SpriteBreakEvent

struct pdg::SpriteBreakEvent

a Sprite joint is breaking because it was overstressed (Chipmunk Physics Only)

This event is generated by a Sprite whenever the forces acting on a joint with another Sprite are greater then the breaking force of the joint. To prevent the break from happening, return true from the handler that gets this event. If you return false then the joint will go ahead and break. 

{
    emitter: {},            // the emitter that generated this event
    eventType: 21,          // the event type (eventType_SpriteBreak)
    action: 13,             // what happened (action_JointBreak)
    actingSprite: {},       // the Sprite to which the forces were applied
    inLayer: {},            // the SpriteLayer that contains the Sprite
    targetSprite: {},       // the Sprite that was previously joined
    impulse: 29.35,         // the magnitude of the impulse that broke the joint
    force: 384.0,           // the force of the collision
    breakForce: 100.0,      // the maximum force the joint could stand before breaking
    joint: {}               // the CpConstraint that defines the joint
}

a Sprite collided with something (Optional)

This event is generated by a Sprite whenever it hits something. If it hits another Sprite action will be pdg.action_CollideSprite, if it hits a wall it will be pdg.action_CollideWall. 

{
    emitter: {},            // the emitter that generated this event
    eventType: 20,          // the event type (eventType_SpriteCollide)
    action: 0,              // what happened (action_CollideSprite or action_CollideWall)
    actingSprite: {},       // the moving Sprite
    inLayer: {},            // the SpriteLayer that contains the Sprite
    targetSprite: {},       // the Sprite that was collided with (for action_CollideSprite)
    normal: {               // the normal vector for the collision
      x: 1.0,
      y: 0.0
    },
    impulse: {              // the impulse imparted by the collision
      x: 29.35,
      y: 0.883
    },
    force: 384.0,           // the force of the collision
    kineticEnergy:883,      // the total kinetic energy of the collision

    arbiter: {},            // the Chipmunk Collision Arbiter (*Chipmunk Physics Only*)

    collisionName: "box1",      // name of collision box that was hit (null for non-spriter)
    withCollisionName: "box2",  // name of other sprite's collision box (null for non-spriter)
    isFirstContact: true        // true if this is the first contact in a collision sequence
}

◆ pdg::SpriteLayerEvent

struct pdg::SpriteLayerEvent

something happened to this layer

This event can be generated with pdg.action_LayerFadeInComplete or pdg.action_LayerFadeOutComplete when the layer finishes a fade.

This event can be generated with pdg.action_ZoomComplete with the layer finishes a zoom animation.

These events are also generated regularly by a SpriteLayer to give the application a chance to take action at various stages of sprite animation and rendering. 

{ 
    emitter: {},            // the emitter that generated this event
    eventType: 18,          // the event type (eventType_SpriteLayer)
    action: 0,              // one of: action_AnimationStart/Complete, etc...
    actingLayer: {},        // the SpriteLayer that is taking action
    millisec: 1025448321,   // the millisecond time when this entire animation step started (when action_AnimationStart fired) 
}

A single step of animation will generate these events:

  • pdg.action_AnimationStart with actingLayer set to the first layer (Z-order = 0)
  • for each layer, with actingLayer set to that layer, in Z-order (0...n):
    • pdg.action_PreAnimateLayer just before animation starts
    • pdg.action_PostAnimateLayer after all animation is complete
  • pdg.action_AnimationComplete with actingLayer set to the last layer

In a similar fashion, a rendering a single frame will generate these events:

  • pdg.action_ErasePort with actingLayer set to the first layer (Z-order = 0)
  • for each layer, with actingLayer set to that layer, in Z-order (0...n):
    • pdg.action_PreDrawLayer just before layer rendering starts
    • pdg.action_PostDrawLayer after all sprites in the layer have been drawn
  • pdg.action_DrawPortComplete with actingLayer set to the last layer
See also
pdg.eventType_SpriteLayer

struct pdg::SpriteTouchEvent

the user clicked on (or touched) a Sprite

This event is generated by a Sprite whenever a user clicks on it (or touches it in on a touch device).

{
    emitter: {},            // the emitter that generated this event
    eventType: 19,          // the event type (eventType_SpriteTouch)
    shift: false,           // true if the shift key is held down
    ctrl: false,            // true if the control key is held down
    alt: false,             // true if the alt (or option) key is held down
    meta: false,            // true if the meta (windows or command) key is held down
    mousePos: {
        x: 456,             // the x coordinate of the mouse when the event occurred
        y: 201              // the y coordinate of the mouse when the event occurred
    },
    leftButton: false,      // true if the left mouse button was down
    rightButton: false,     // true if the right mouse button was down
    buttonNumber: 0,        // button number (0=left, 1=right, 2=middle) or finger ID for touch
    lastClickPos: { 
        x: 456,             // the x coordinate of the last mouseDown event
        y: 201              // the y coordinate of the last mouseDown event
    },
    lastClickElapsed: 201,  // the number of milliseconds since the last mouseDown
    touchType: 22,          // touch_MouseDown(22), touch_MouseUp(23), or touch_MouseClick(24)
    touchedSprite: {},      // the Sprite that was clicked or touched
    inLayer: {}             // the SpriteLayer that contains the Sprite
}

A single click may generate up to three separate events, each with a different touchType:

  • pdg.touch_MouseDown when a mouse button goes down inside a Sprite
  • pdg.touch_MouseUp when a mouse button is released inside a Sprite
  • pdg.touch_MouseClick when the mouse button was clicked and released inside the same Sprite
Note
Click events must be turned on for each Sprite using Sprite.setWantsClickEvents()
If no Sprite was hit by a mouse event, the original pdg.eventType_MouseDown or pdg.eventType_MouseUp is passed on through the EventManager
Bug
will emit touch_MouseClick event if one button was clicked inside a sprite and a different button then released in the same Sprite.

the user pushed down on a key

To detect the key being released, use eventType_KeyUp. For keyboard data entry, use eventType_KeyPress.
